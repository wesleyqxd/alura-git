############################################################
## LISTA DE COMANDOS DO CURSO INICIAL DO GIT E GITHUB      #
###########################################################

Iniciar um repository git : git init
Verificar status do repositorio : git status
Pedir pro git monitorar arquivo: git add <file>
Adicionar modificar em um arquivo : git add <file>
Verificar em qual branch estou : git log
configurar pra um repositorio apenas : git config --local user.name wesleyfurnas
configurar pra um repositorio apenas : git config --local user.email wesleyfurnas35@gmail.com
Verificar alteracoes entre commits : git log -p 
Visualizar informacoes de forma resumid entre commits : git log --oneline 
Criar repositorio puro que so armazena as alteracoes nos arquivos e nao os arquivos fisicos propriamente ditos: git initi --bare 
Git remote lista todos os repositorios remotos que o repositorio atual conhece 
git remote add local path/past/server_url_past_etc faz repositorio atual enchegar o repositorio remoto path/.. que é renomeado para local  
verificar caminho do servidor remoto : git remote -v
clonando um repositorio pra uma pasta especifico : git clone path/repositorio/ name_do_novo_projeto
Fazendo upload para o servidor remoto: git push servidor_destino brach_servidor_destino
Fazer download de arquivos de repositirio remoto: git pull _repositorio_remoto nome_da_branch
estou no client2
Adicionar modificacoes de uma brach na brach principal : git checkout master
Adicionar modificacoes de uma brach na brach principal : git merge _brach_com_modificacoes
visualizar linnhas de desenvolvimento git: git log --graph
adicionar todos os commits do brach de desenvolvimento ao brach principal: git rebase _brach_de_desenvolvimento
 sempre antes de fazer modificacoes dar git pull _servidor_ master 
atribuir nome de usuario ao commit: git config --local user.name: name_user
desfazer alteracoes antes de dar commit : git checkout -- nome_do_arquivo
desfazer alteracoes apos git add  : git reset HEAD nome_do_arquivo
desfazer commit  : git revert hash_do_commit
salvar arquivos pra local temporario sem dar commit   : git stash
aplicar os arquivos salvos temporariamente ao branch de trabalho  : git stash {numero_do_stash}
verificar a lista de stash : git stash list
aplicar e já remover stash : git stash pop
navegar a qualquer ponto de desenvolvimento: git checkout commit_number
Sempre que navegador a um commit, é importante criar um novo branch pra não perder eventuais commits criados naquele ponto de desenvolvimento
diferença entre commits: git diff num_commit1.. numcommit
Adicionar uma tag para gerar versao: git tag -a nome_Da_tag -m Lançando primeira versão...
comando pra adicionar tag no servidor remoto: git push servidor _nome_da_tag_ 
para contribuir em projetos aos quais nao é dono, basta fazer um fork no projeto e copiar a url do servidor
adiciona o servidor remoto na lista de servidores remotos : git remote add endereço_do_servidor_da_fork
basta  fazer push no servidor da fork e vai em pull request no github eescolhendo os branchs corretos dos projetos  
clicar em create pull request e adicionar um comment   
agora vai no servidor principal do projeto e aprova o pull request
